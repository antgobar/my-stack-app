StackCX Technical Assessment Brief
Introduction
Your task is to build a music player web app (like Spotify) in two
hours. Please submit your solution via email within two hours of
receiving this brief.
You have been provided with a small sample of audio tracks to use as
test data.

Scope
It is obviously impossible to build Spotify in 2 hours! Below we
have provided some core requirements to implement. When you’ve met
those there are a set of aspirational requirements that you could
consider, but you are also generally free to spend time writing
interesting features or maturing the technical solution.

Core Functional Requirements
User Interface
• Display a list of tracks (title, artist, album cover).
• Show currently playing track with title and artist.
• Play/pause functionality.
• Skip to next/previous track.
Audio Playback
• Stream the included audio tracks.
• Volume control.
Track Interaction
• Allow the user to select a track to play.

Aspirational Functional Requirements
These are just ideas, after meeting the core requirements your free
to chose some of these, your own ideas or work on other technical
aspects.
• Add, remove and view favourite tracks.
• Create and add tracks to playlists.

• Display track progress (seek bar).
• Shuffle mode.
• Add tracks to queue.
• View recently played track.
• Searching and filtering tracks.
• Remember and persist session information (which track is
playing and where)

Hints
Prioritise build something simple that works end to end. For
example, just keeping the audio as part of the application rather
than hosting it. And then build out complexity from that.
Commit and push regularly. You’re welcome to include a finished
solution, but also with an open unfinished branch of something you
were working on but didn’t finish.
